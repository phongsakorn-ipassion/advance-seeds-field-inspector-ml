# Proposal — Adopt OpenSpec Governance

## Why

This ML repo will produce model and calibration artifacts consumed by the
Advance Seeds Field Inspector demo app. The work will evolve through dataset
validation, segmentation training, calibration validation, and mobile export.
Those changes need a durable source of truth outside chat history so future AI
and human work stays aligned.

## What Changes

- Add OpenSpec project configuration under `openspec/config.yaml`.
- Add canonical specs for:
  - `project-governance`
  - `dataset-preparation`
  - `segmentation-training`
  - `calibration-validation`
  - `mobile-model-export`
- Add Codex OpenSpec skills through `openspec init --tools codex`.
- Document that every non-trivial future change must update OpenSpec before it
  is considered complete.

## Capabilities

### New Capabilities

- `project-governance`: OpenSpec rules, validation, and workflow expectations.
- `dataset-preparation`: dataset config and YOLO segmentation label validation.
- `segmentation-training`: model training and metric requirements.
- `calibration-validation`: px/mm conversion and measurement-error targets.
- `mobile-model-export`: app artifact and metadata handoff contract.

### Modified Capabilities

None. This is the first OpenSpec adoption for this repo.

## Non-goals

- Training a model in this change.
- Changing the consuming mobile app.
- Moving client datasets or model weights into Git.
- Replacing the existing Python helper scripts.

## Impact

- Adds `openspec/` as the spec source of truth.
- Adds `.codex/skills/` generated by OpenSpec for the Codex workflow.
- Updates project documentation to require OpenSpec updates for future work.
